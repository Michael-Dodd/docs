Loqate for Salesforce Setup and Usage

 

Setup

 

Setup for the Loqate app follows the following steps:

 

1.      Install the app from the Salesforce App Exchange

2.      Assign administrator and user permission sets

3.      Add your Loqate API Key

4.      Setup Address Mappings

5.      Add Loqate widgets to your object pages

 

 

Installing the App

 

The app can be installed via the Salesforce App Exchange listing. Adding the app from here will ensure you are always installing the latest version of our app.

 

Assigning Administrator and User permission sets

 

The Loqate app includes two permission sets Loqate Administrator and Loqate User. The use cases for these sets are as follows:

 

       Loqate User

o   This permission set is for any user who needs to use the Loqate controls on the page, to validate addresses, email addresses, phone numbers, but dont need to administer the settings of the app itself.

       Loqate Administrator

o   Users with this permission set will be able to use the controls in the same way as those users who have the Loqate User permission set.

o   Users with the Loqate Administrator permission set will also have access to the Loqate settings tab and page under the Loqate App, and will be able to administer settings and mappings for the app.

 

Setup the App

 

Once the app is installed and permission sets assigned, administrator users will be able to access the settings page under the Loqate App.

 

 

You will need to create an API key via your Loqate account. Once you have this API key you can enter it in the API Key field and save your settings.

 

Note: Your API key is designed to be an open access key to your account, there are security controls within your Loqate account to ensure this is not abused. The key used in the app will be your primary mechanism of reporting for usage, so you should create a new key rather than re-using one you may already have on a website or other application.

 

With your key in place and your settings saved you will be able to use the email and phone validators. In order to use the Address Validator you will need to add object Mappings.

 

Creating address mappings

 

You should create an address mapping for each address you want to be able to populate from the address validation control. This might involve creating multiple mappings for a single object. For example, for the Contact object you might create one mapping for the Mailing address and one for the Other address.

 

Follow the following steps to create a mapping:

 

1.      From your Loqate settings screen select New Mapping.

2.      Give your mapping a distinct and memorable name, i.e. Contact Mailing

3.      Select the object you want to create a mapping for, in our example well choose Contact.

4.      Select + Add Field.

5.      Choose a field on your chosen object that you want to populate with address data, i.e. MailingPostalCode.

6.      Select a Loqate element that will be used to populate the above field when an address is selected. i.e. postalCode

a.      You can also use custom field mappings here to do so select custom format from the element list. This will cause a custom pattern field to appear.

b.      Enter your custom field mapping for example {line1}{, {line2}}. This pattern will concatenate line1 and line2 with a , if line2 is available and just line1 if not. For more information on these custom formats, see the Formatting fields section of https://www.loqate.com/developers/guides/advanced-setup-guide/.

7.      Select Add Field Mapping to add your new mapping to the mappings list.

8.      Continue to add fields for each element you want to populate with a selected address, when you have finished mapping your fields, select Save.

 

Repeat the above process for any more addresses for which you want to create mappings.

 

 

 

 

Add Loqate widgets to your object pages

 

 

In order to use the controls available via the Loqate app, you need to add them to your object pages, in this example well use the contact object.

 

From the contact details page, select Settings -> Edit Page.

 

 

Using the lightning app builder, you can drag the Loqate controls into your page. The controls will be listed in the component list under Custom Managed.

Address Finder Settings

 

There are some additional settings available for the address finder control.

 

 

       Field Mappings

o   By default the Address finder control will display a select box listing all mappings you have created for the object on which the control has been placed. You can then select the mapping to which you want to apply the address you select.

o   If you wish you can lock the address finder to a particular mapping, to do so, enter the name of the mapping in the Field Mappings box in the address finder options. The control will no longer display the select box and will instead always use the mapping you have chosen.

       Address Title Prefix

o   Text entered in this option ill be prepended to the title of the address finder widget. This can be useful when you are locking an address finder to a particular mapping and might have multiple address finders on the page.

o   For example you might want two address finders on the contact page, one for mailing and one for other. By entering Mailing in this box the title will read Mailing Address Finder, helping to distinguish it from other address finders.
